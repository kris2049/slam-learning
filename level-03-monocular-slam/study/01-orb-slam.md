8|# 第1部分: ORB-SLAM — 现代特征法 SLAM 的标杆
9|
10|## 1.1 从 PTAM 到 ORB-SLAM
11|
12|### PTAM (Parallel Tracking and Mapping) — 2007, Klein & Murray
13|
14|**革命性贡献**：将 SLAM 分为两个并行线程。
15|
16|```
17|┌─────────────────┐     ┌─────────────────┐
18|│  Tracking 线程   │     │  Mapping 线程    │
19|│  (每帧运行)      │ ←── │  (关键帧触发)    │
20|│  • 特征匹配      │     │  • 局部 BA       │
21|│  • 位姿估计      │     │  • 新地图点生成   │
22|│  • 关键帧决策    │     │  • 地图点剔除     │
23|└─────────────────┘     └─────────────────┘
24|```
25|
26|**PTAM 的局限**：
27|- 需要手动初始化（必须在平面上移动相机）
28|- 只能处理小场景
29|- 容易丢失（没有重定位）
30|
31|### ORB-SLAM — 2015, Mur-Artal et al.
32|
33|**三条线程，完整系统**：
34|
35|```
36|        ┌──────────────┐
37|  图像 →│   TRACKING   │→ 位姿
38|        └──────┬───────┘
39|               │ 新关键帧
40|        ┌──────▼───────┐
41|        │ LOCAL MAPPING│→ 局部地图
42|        └──────┬───────┘
43|               │ 回环候选
44|        ┌──────▼───────┐
45|        │ LOOP CLOSING │→ 全局修正
46|        └──────────────┘
47|```
48|

51|## 1.2 Tracking 线程 — 每帧的位姿估计
52|
53|### 模式一：运动模型跟踪
54|
55|用上一帧的**速度**预测当前位姿 → 在当前位姿附近做特征匹配 → 优化。
56|
57|### 模式二：参考关键帧跟踪
58|
59|用最近关键帧的特征 → 在当前帧搜索匹配 → PnP 求解位姿。
60|
61|当运动模型不可靠时使用（如算法刚启动，或刚重定位后）。
62|
63|### 模式三：重定位 (Relocalization)
64|
65|位姿丢失时：
66|1. 用 BoVW 搜索相似关键帧
67|2. 对每个候选，RANSAC + PnP 验证
68|3. 如果找到足够内点 → 恢复位姿
69|
70|### 新关键帧插入条件
71|
72|必须同时满足以下条件之一：
73|
74|1. **距离上**：与上一关键帧平移距离 > 阈值（如场景深度的 5%）
75|2. **角度上**：与上一关键帧旋转角度 > 阈值
76|3. **跟踪质量**：跟踪到的地图点 < 阈值（场景变化大）
77|4. **时间上**：距离上一关键帧 > 一定帧数
78|
79|且必须保证当前帧有足够的跟踪点（防止插入质量差的关键帧）。
80|

83|## 1.3 Local Mapping 线程 — 局部地图管理
84|
85|### 局部 BA
86|
87|优化：**当前关键帧 + 共视最强的邻居**（~20-30个关键帧）+ 它们观测到的所有地图点。
88|
89|$$\min \sum_{\text{local KFs} \times \text{local MPs}} \|\pi(T_{KF}, MP) - z\|^2$$
90|
91|**为什么是局部？** 新关键帧只影响它附近的地图，远距离不受影响。
92|
93|### 地图点剔除 (Map Point Culling)
94|
95|新生成的地图点在接下来3帧中必须通过检验：
96|- 必须在**≥25%**的预期可见帧中被找到
97|- 如果连续3帧通过率 < 25% → 剔除
98|
99|这确保了地图点质量——不可靠的点被及时清理。
100|
101|### 新地图点生成
102|
103|在**当前关键帧 + 最佳共视关键帧**之间做三角化：
104|1. 找未匹配的特征（在两个关键帧中都有，但不在同一地图点）
105|2. 对极几何验证（沿对极线搜索匹配）
106|3. 三角化生成3D点
107|4. 检查：正深度、重投影误差<阈值、尺度一致性
108|
109|### 冗余关键帧剔除
110|
111|如果一个关键帧中 90% 的地图点已被 ≥3 个其他关键帧观测到 → 删除该关键帧（信息冗余）。
112|

115|## 1.4 Loop Closing 线程 — 回环检测与修正
116|
117|### 回环检测三步走
118|
119|**步骤1: 回环候选检测**
120|
121|当前关键帧的 BoVW 向量在数据库中搜索 → Top-N 候选 → 排除最近的关键帧 → 保留分数 > 阈值的。
122|
123|**步骤2: 计算 Sim(3) 变换**
124|
125|单目 SLAM 中有**尺度漂移**（长时间运行后尺度会变化）→ 用 Sim(3)（7自由度：R, t, s）而非 SE(3) 做对齐。
126|
127|- 对候选关键帧做 ORB 匹配 + RANSAC
128|- 如果有足够内点 → 估计 Sim(3) 变换
129|- 验证：重投影误差检查
130|
131|**步骤3: 回环修正**
132|
133|1. **Pose Graph Optimization**：以回环边为约束，优化**所有关键帧的位姿**（不优化路标）
134|2. **全局 BA**（单独线程，慢）：同时优化所有关键帧 + 所有地图点
135|
136|### 本质图 (Essential Graph)
137|
138|位姿图优化的图结构 = 共视图的生成树 + 共视权重 > 100 的边 + 回环边。
139|
140|比完整共视图稀疏 → PGO 更快。
141|

144|## 1.5 共视图 (Covisibility Graph) — 核心数据结构
145|
146|每个关键帧记录它观测到的地图点。
147|
148|两个关键帧**共视** = 它们共享的地图点数 ≥ 阈值。
149|
150|```
151|KF₁ ───(80)─── KF₂
152| │              │
153|(40)          (60)
154| │              │
155|KF₃ ───(50)─── KF₄
156|```
157|
158|边权重 = 共视地图点数。
159|
160|**共视图的三个用途**：
161|1. **Local BA**：选择共视最强的邻居
162|2. **位姿图**：构建 Essential Graph
163|3. **回环**：找到与当前帧无关的候选
164|

167|## 1.6 自动初始化 (ORB-SLAM 的关键创新)
168|
169|单目 SLAM 的初始化：从**无到有**创建初始地图。
170|
171|**并行计算两个模型**：
172|
173|$$x_{H} = \frac{\#\text{inliers}_H}{\#\text{inliers}_H + \#\text{inliers}_F}$$
174|
175|$$\text{if } x_H > 0.4 \text{: choose homography (planar)}$$
176|$$\text{: else choose fundamental (general)}$$
177|
178|选择后：
179|1. 从选中的模型恢复 (R, t)
180|2. 三角化初始地图点
181|3. 全局 BA（只优化初始的几张关键帧）
182|4. 从初始地图点估计场景中值深度 → 所有阈值基于此缩放
183|

186|## 1.7 地图点管理策略总结
187|
188|| 操作 | 触发条件 | 目的 |
189||------|----------|------|
190|| **生成** | 新关键帧与共视帧三角化 | 扩展地图 |
191|| **融合** | 两个地图点投影非常接近 | 消除重复 |
192|| **剔除** | found_ratio < 0.25 | 去掉不可靠的点 |
193|| **替换** | 新地图点比旧的有更多观测 | 提高质量 |
194|| **BA 优化** | 局部/全局 BA | 精化位置 |
